SHARE-INSIGHTS · v0.1

Turn any AI session into a clean, shareable URL

Ask your AI to summarize what you just did, and publish the result to a short URL like share-insights.pages.dev/a1b2c3d4e5f6. No accounts. No tracking. Auto-deleted after 90 days. Works for Claude, ChatGPT, Cursor, Codex.

1Install for your AI tool

Pick the tool you use. All paths end at the same backend; the URL format is identical.

🤖

Claude Desktop free, all plans

Works on the Claude desktop app for Mac & Windows.

Open (or create) ~/Library/Application Support/Claude/claude_desktop_config.json and add a share-insights entry under mcpServers:

{
  "mcpServers": {
    "share-insights": {
      "url": "https://share-insights-mcp.tshradheya.workers.dev/mcp"
    }
  }
}

Quit and relaunch Claude Desktop. In any chat say "share this conversation" — Claude will call the prepare_insights tool, show you what's about to publish, ask you to confirm, then call publish_insights.

2Don't want to install anything? Use any AI as-is

Works for free ChatGPT, Claude.ai (any plan), Gemini, anything. Three steps:

  1. Paste this prompt at the end of your AI conversation:
Summarize this conversation into a self-contained HTML document I can publish.

Required structure: sections for TL;DR, Decisions, Findings, Code & diffs, Open Questions, Next Steps. Include only sections that have real content.

REDACT aggressively before writing: replace anything that looks like an API key (AKIA…, ghp_…, sk-…, sk-ant-…, AIza…, xox*-…), JWT (eyJ…eyJ…), password= or Authorization: Bearer values, paths under /Users/me/ or /home/me/, internal hostnames, customer/account names not already public, and personal email addresses. Note every redaction in a "Redactions" section at the bottom.

The HTML must be a complete document. Do NOT include <script>, <iframe>, external CSS/JS, <link>, <form>, or any on* event attributes — the publisher rejects them. Diagrams must be inline SVG. Images must be base64 data: URIs. Use embedded CSS only.

Output ONLY the HTML, starting with <!doctype html> — no commentary before or after.
  1. Copy the entire HTML the AI gives you.
  2. Paste into share-insights.pages.dev/new (publish form — coming with v0.2) or, for now, run via the API:
curl -X POST https://share-insights.tshradheya.workers.dev/api/publish \
  -H 'content-type: application/json' \
  -H 'user-agent: share-insights-manual/0.1' \
  --data @- <<EOF
{"html": $(jq -Rs . < your-file.html), "title": "your title"}
EOF

3Ask your AI to install it for you

If you're already in a chat with Claude, ChatGPT, etc., paste this and the AI will walk you through the right install for the tool you're in:

I want to install the share-insights MCP server so I can publish summaries of our conversations to share-insights.pages.dev.

Please:
1. Detect which AI tool I'm using (you're it).
2. Tell me the exact file to edit OR setting to change for THIS tool.
3. Give me the snippet to paste:
   - MCP URL: https://share-insights-mcp.tshradheya.workers.dev/mcp
   - The MCP exposes two tools: prepare_insights(html, title?) and publish_insights(preview_token).
4. Walk me through it step by step. Ask one question at a time.
5. After install, test it by asking me what we just worked on, summarize it, redact secrets, show me a preview, ask permission, then publish.

4Privacy & limits

  • URLs are unguessable 12-char content hashes. No listing page, not indexed (noindex + robots.txt: Disallow: /).
  • Pages auto-delete 90 days after the last view.
  • No accounts, no identifiers. Only the rendered HTML + minimal metadata.
  • Anyone with the URL can read the page. No per-page password in v0.1.
  • 10 MB max HTML, 20 publishes per IP per day, 5 per hour.
  • Every page has a Report this page footer link reviewed within 24h. Full policy →
