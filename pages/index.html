How it works

Three layers of defense before anything goes live

An anonymous publishing endpoint is a credential-leak engine without guardrails. We have three.

Agent redacts in chat

The skill / GPT / MCP tool prompts the model to strip API keys, JWTs, home directory paths, customer names — and produce a "Redactions" list so you can see what was removed.

You confirm the preview

The agent shows you the rendered preview in the chat itself and asks "publish? yes/no". Nothing is uploaded without your explicit yes.

Server has the final say

A regex secret scanner + strict HTML allow-list (no <script>, no external resources, no on* handlers) runs server-side before the page touches R2.

Privacy

Made to be safe by default

Unguessable URLs

12-char content hash. No listing endpoint exists.

Not indexed

noindex meta + robots.txt: Disallow: /.

90-day TTL

Auto-deleted after 90 days of no views.

No accounts

No login, no tracking. Only a salted IP hash for rate-limiting.

Open source

MIT-licensed. Run it yourself if you want.

One-click takedown

Every page has a Report link. Reviewed within 24h.
